## 🛠️ Technical Decisions & Tradeoffs

### Why This Approach?

1. **Test-First Development**
   - Every utility tested before use
   - Real-world edge cases covered (JWT tokens, SAML redirects)
   - Fast unit tests (no network, no credentials)

2. **Layered Architecture**
   - Phase 1: Session capture (browser automation)
   - Phase 2: API discovery (network sniffing)
   - Phase 3: CLI implementation (simple wrapper around HTTP)
   - Each layer independently testable

3. **Offline-First Testing**
   - All unit tests mock HTTP responses
   - No credentials needed for tests
   - Fast feedback loop (1-2 seconds)
   - CI/CD ready

4. **Okta SSO Support**
   - Headless browser for MFA support
   - Automatic SAML redirect handling
   - Cookies captured via Playwright API (not JavaScript)

### Why Not Alternatives?

- ❌ **Using public Zoom API**: No public API exists for this use case
- ❌ **Web scraping**: Zoom is a SPA; all data via internal JSON APIs
- ❌ **Hardcoding credentials**: Not secure; SSO is required anyway
- ❌ **Custom auth**: Okta already handles everything; just capture the result
